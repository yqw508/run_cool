# 萌系 2D 美术升级设计

## 目标

把当前混合素材和原型素材的视觉效果，升级为统一、精美、萌系的 2D 手机跑酷游戏风格。目标是让大厅、地图选择、跑酷背景、障碍物、收集物和角色 PNG 看起来属于同一个高质量儿童向休闲游戏。

## 美术标准

- 风格：精致萌系 2D 插画，圆润造型，柔和光照，轮廓清晰。
- 受众：儿童和亲子向休闲玩家。
- 色彩：明亮、柔和、糖果色，但不能单一刺眼；以草绿色、天空蓝、暖黄色、粉色点缀为主，并保证游戏可读性。
- 细节：比图标包和占位素材更丰富，但跑酷中不能干扰障碍物、收集物和角色识别。
- 透视：大厅和地图页以正向 2D 展示为主；跑酷页需要支持中间道路向远处延伸的三车道视觉。
- 一致性：所有核心背景应该像同一款游戏的美术，而不是不同素材包拼出来的画面。

## 资产策略

开源素材包仍然可以用于临时占位、小图标、临时收集物和早期障碍物尺寸验证。但它们不再作为最终背景图的主要来源，因为目前找到的免费素材整体太简陋，不符合“精美、萌系、儿童跑酷”的目标。

最终核心场景应该使用统一风格生成或定制的 PNG：

- 角色大厅花园背景。
- 地图选择页世界背景。
- 每个跑酷地图一张主背景。
- 每个地图一张缩略图，可由主背景裁切或单独生成。
- 每个地图对应一组高质量障碍物 PNG。
- 高质量收集物 PNG，用于跑酷中的连续收集物串。

## 第一批美术基准图

不要一开始就批量生成全部地图。先生成并确认 3 张基准图：

1. 角色大厅花园背景。
2. 地图选择页背景。
3. 第一个跑酷地图背景，优先选择花园或校园。

这 3 张的风格通过后，再批量生成其它地图、障碍物、收集物和缩略图。

## 页面要求

### 角色大厅

角色大厅要像“角色们站在一个可爱的花园里”。背景可以包含花草、草地、圆润树木、小路、阳光、云朵和一些精致的花园装饰。画面中间和底部需要留出足够干净的空间，用来放角色 PNG 和底部操作按钮。

角色使用静态正面 PNG，只做轻微待机动效，例如小幅上下浮动、轻微旋转、闪光点缀。不要再做拉伸、挤压、肢体拆分动画，避免角色看起来散架。

### 地图选择页

地图选择页要像一个可爱的童话世界地图或花园路线板。每个地图节点都要有缩略图，缩略图应该预览真实跑酷地图环境。地图节点需要是清晰的大触控目标，并且不同地图之间有明显视觉区分。

### 跑酷页

每个跑酷地图都需要一张完整背景，并服务当前三车道跑酷玩法。背景不能抢障碍物和收集物的注意力。中间道路区域要保持清晰，装饰细节主要放在两侧和远景。

收集物应该以更长的连续串出现，对标主流跑酷游戏的奖励节奏。收集物串要在高速移动时也清楚、好吃、让人有收集欲。

## 初始提示词

### 角色大厅花园背景

生成一张精美萌系 2D 手机游戏角色选择大厅背景。画面是明亮的魔法花园，有柔软绿色草地、圆润树木、彩色花朵、一条小路、温暖阳光和轻柔云朵。儿童向、可爱、高质量休闲游戏美术、柔和光照、干净形状、柔和糖果色，细节丰富但不要杂乱。中间和下方需要留出空间摆放五个角色 PNG 和 UI 按钮。不要角色、不要文字、不要 logo、不要 UI。竖版手机 9:16 构图。

### 地图选择页背景

生成一张精美萌系 2D 手机游戏地图选择页背景。画面像一个童话故事书里的世界地图，处在明亮花园公园中，有一条柔和小路连接不同主题区域，例如花园、校园、糖果商场、动物园、游乐园。圆润造型、儿童向配色、高质量休闲游戏 UI 背景、构图清晰可读、柔和阳光、柔和糖果色。画面需要给大地图节点和标题留空间。不要角色、不要文字、不要 logo、不要 UI。竖版手机 9:16 构图。

补充硬性要求：不要画手机外框、不要系统状态栏、不要地图名称文字、不要地点标签、不要按钮、不要卡片、不要水印。只生成纯背景环境和柔和路径，地图节点、标题和文字由游戏代码绘制。

### 花园跑酷背景

生成一张精美萌系 2D 无尽跑酷游戏背景，主题是魔法花园。竖版手机 9:16 构图，中间有清晰的跑酷道路或小径向远处延伸，需要支持三车道玩法可读性。道路两侧有柔软草地、花朵、圆润树木和花园装饰，温暖阳光，轻微景深，高质量可爱 2D 休闲游戏美术。中间道路要干净，方便放置障碍物、收集物和玩家角色。不要角色、不要文字、不要 logo、不要 UI。

补充硬性要求：不要画任何障碍物、不要画收集物、不要画道具、不要画木栏、不要画路障、不要画金币、不要画角色、不要水印。道路中间必须保持干净，只保留跑酷道路和两侧环境装饰，真实障碍物和收集物由游戏代码生成。

## 负面提示词

避免照片写实、3D 渲染、黑暗氛围、恐怖感、真实人物、文字、水印、logo、UI 面板、模糊细节、杂乱堆砌、过于扁平的图标包风格、低分辨率、硬边阴影、透视变形、物体遮挡中间跑酷道路。

## 接入说明

- 最终确认的背景图放到 `src/assets/backgrounds/`。
- 地图缩略图放到 `src/assets/maps/thumbnails/`。
- 最终障碍物 PNG 放到 `src/assets/obstacles/`。
- 最终收集物 PNG 放到 `src/assets/collectibles/`。
- 当前开源素材只作为临时 fallback，最终美术完成前可以继续占位。
- 不再重新引入 Three.js；整体方向是纯 2D PNG。

## 验收标准

- 角色大厅首屏一眼能看出是精美萌系花园场景。
- 地图选择页缩略图和真实跑酷地图风格一致。
- 跑酷背景在手机视口下不影响三车道可读性。
- 角色 PNG 不再通过拉伸或拆肢体动画表现动作。
- 开源占位背景替换掉之后，才能认为美术升级完成。
