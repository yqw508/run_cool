# 萌娃乐园与萌娃连连看 MVP 设计

## 背景

当前项目以“萌娃酷跑”为核心 H5 游戏，已经有五个萌娃角色、角色选择大厅、酷跑玩法，以及校园、商场、动物园、游乐园四个主题关卡。后续产品定位升级为面向小学生的寓教于乐产品，“萌娃酷跑”只是“萌娃乐园”中的一个游戏入口。

本设计新增“萌娃连连看”作为第二个核心游戏。虽然产品名称叫连连看，第一版玩法采用更适合移动端和小学生反馈节奏的三消式交换：相邻方块交换，横向或纵向三个相同角色合成并触发动画。

## 目标

- 将现有单游戏入口升级为“萌娃乐园”首页，承载酷跑、连连看、诗集入口。
- 新增“萌娃连连看”MVP，复用酷跑五个角色作为三消方块。
- 复用酷跑四个主题的视觉语言和主题配置，形成统一世界观。
- 在结算阶段引入古诗诗卡和收藏，形成轻量学习闭环。
- 为后续朗读、跟读、背诵、更多诗库和成长体系预留接口，但不在第一版实现复杂学习系统。

## 非目标

- 第一版不做账号、云端进度、家长端、排行榜或付费体系。
- 第一版不做语音识别、跟读评分或真实朗读音频播放。
- 第一版不重做酷跑核心玩法。
- 第一版不把酷跑场景截图作为连连看底图；连连看应复用主题配置并单独绘制适配棋盘的背景。

## 产品结构

### 萌娃乐园首页

首页替代当前单一游戏入口，展示三个主要入口：

- 萌娃酷跑：进入现有酷跑玩法。
- 萌娃连连看：进入三消式角色合成玩法。
- 诗集：查看已解锁和已收藏的主题诗卡。

首页应保留当前萌娃角色和花园/乐园氛围，让用户感知这是同一个产品世界。

### 萌娃连连看入口

连连看入口包含主题关卡选择。第一版沿用酷跑四个主题：

- 校园
- 商场
- 动物园
- 游乐园

每个主题拥有相同的三消核心规则，但背景、关卡目标、诗库和结算文案不同。

## 核心玩法

### 棋盘规则

- 棋盘由五种角色方块组成，对应酷跑已有五个角色。
- 玩家可交换相邻的两个方块。
- 交换后如果形成横向或纵向三个及以上相同角色，则消除/合成成功。
- 无法形成三连的交换应回弹。
- 消除后上方方块下落，空位补入新方块。
- 支持连续消除和连击计数。

### 角色方块与合成动画

五个角色沿用酷跑现有角色设定。每种角色三连时触发专属动画：

| 角色 | 三连动画 |
| --- | --- |
| 小弟弟 | 喝奶瓶 |
| 小星星 | 大西瓜 |
| 冰雪公主 | 钻石 |
| 小布丁 | 布丁果冻 |
| 空军上尉 | 飞机飞过 |

动画可以先用 Phaser 2D 图形、粒子、图片缩放/飞行动效实现。后续如果 3D 角色资产成熟，再替换为 3D 或更丰富动画。

### 得分与星级

一局结束后根据以下因素计算分数和星级：

- 基础三连次数
- 四连、五连或更多连消奖励
- 连击奖励
- 剩余步数或剩余时间奖励

第一版建议采用三星结算：

- 1 星：完成基本目标
- 2 星：达到中等分数
- 3 星：高分或高连击

星级用于解锁主题诗库中的诗卡。

## 主题关卡

### 复用原则

现有酷跑主题由 `src/game/levels.ts` 配置，包括主题 id、颜色、地标、障碍物和音乐 key。连连看应共享主题 id、主题色和地标概念，但使用适合棋盘的绘制方式。

推荐抽出共享主题数据，避免酷跑与连连看各维护一套互相漂移的主题定义。

### 主题映射

| 主题 | 连连看呈现 | 学习方向 |
| --- | --- | --- |
| 校园 | 教学楼、操场、图书馆氛围，棋盘居中 | 入门、熟悉古诗、识字 |
| 商场 | 游戏厅、餐厅、影院氛围，色彩更热闹 | 生活认知、数字、后续生活数学 |
| 动物园 | 动物园门、围栏、动物点缀 | 动物、自然、主题古诗 |
| 游乐园 | 摩天轮、过山车、旋转木马氛围 | 奖励关、综合复习 |

## 古诗诗库

### 诗库规则

每个主题拥有一个诗库。玩家完成该主题关卡后，根据分数和星级解锁主题诗卡。

第一版诗卡包含：

- 诗名
- 作者
- 正文
- 所属主题
- 解锁星级
- 收藏状态
- 后续朗读/跟读入口状态

第一版支持诗卡展示和收藏。朗读、跟读、背诵先保留入口和数据字段，但入口显示为即将开放或不可用状态。

### 示例诗库

第一版可先配置少量示例诗卡，具体诗目可在实现前再校对：

| 主题 | 示例诗卡 |
| --- | --- |
| 校园 | 《静夜思》《登鹳雀楼》《春晓》 |
| 商场 | 先放通用启蒙诗卡，后续扩展生活认知或数字主题内容 |
| 动物园 | 《咏鹅》《绝句》 |
| 游乐园 | 复习型诗卡或奖励诗卡 |

如果某个主题暂时没有强绑定古诗，仍按主题诗库结构实现。第一版每个主题至少配置一张可正式展示的诗卡；诗目不够时减少该主题诗卡数量，不使用会出现在用户界面的临时内容。

## 数据模型

### 角色方块

```ts
type MatchTileKind =
  | 'baby-brother'
  | 'little-star'
  | 'ice-princess'
  | 'pudding'
  | 'air-captain';

type MatchTile = {
  id: string;
  kind: MatchTileKind;
  row: number;
  col: number;
};
```

### 连连看主题

```ts
type MatchTheme = {
  id: ThemeId;
  label: string;
  poemLibraryId: string;
  targetScore: {
    oneStar: number;
    twoStars: number;
    threeStars: number;
  };
};
```

### 诗卡

```ts
type PoemCard = {
  id: string;
  themeId: ThemeId;
  title: string;
  author: string;
  lines: string[];
  unlockStars: 1 | 2 | 3;
  audioUrl?: string;
  readAlongEnabled: boolean;
};
```

### 本地进度

```ts
type BabyParkProgress = {
  unlockedPoemIds: string[];
  favoritePoemIds: string[];
  bestStarsByTheme: Partial<Record<ThemeId, 1 | 2 | 3>>;
  bestScoreByTheme: Partial<Record<ThemeId, number>>;
};
```

第一版可使用 `localStorage` 保存本地进度。后续接入账号后再迁移到服务端。

## 架构建议

### 场景拆分

当前 `src/main.ts` 直接创建 Phaser 游戏并进入 `RunnerScene`。新增 MVP 后，建议引入明确的场景结构：

- `ParkHomeScene`：萌娃乐园首页。
- `RunnerScene`：现有酷跑场景，保留并适配从首页进入。
- `Match3Scene`：萌娃连连看主玩法。
- `PoemBookScene`：诗集与收藏。

如果一次性迁移风险较高，可以先用一个顶层 UI 状态切换方式实现首页，再逐步拆成 Phaser Scene。

### 逻辑模块

连连看玩法逻辑应从渲染场景中拆出纯函数或小模块，便于测试：

- 棋盘初始化
- 判断可交换
- 查找横/竖匹配
- 消除与下落
- 补齐新方块
- 计分与星级
- 解锁诗卡

这些模块应有单元测试，避免后续动画和 UI 改动破坏规则。

## UI 与交互

### 棋盘

- 移动端优先，棋盘位于屏幕中部。
- 方块尺寸稳定，不因角色名称或状态变化造成布局跳动。
- 角色图片来自现有角色资产，必要时做裁切或统一背景框。
- 选中方块有清晰边框或浮起效果。
- 可交换目标高亮，非法交换回弹。

### 结算

结算页展示：

- 本局得分
- 星级
- 三连次数和最高连击
- 解锁的诗卡
- 收藏按钮
- 再玩一局
- 返回主题选择

如果没有解锁新诗卡，展示本主题下一张可解锁诗卡的进度提示。

### 诗集

诗集页展示已解锁诗卡和收藏状态。第一版可按主题筛选，未解锁诗卡显示剪影或锁定态。

## 错误处理与边界

- 棋盘初始化时不能生成开局即无可用交换的死局；如检测到死局则重洗。
- 补齐新方块后应避免无限连锁导致无法结算；连锁可以允许，但要有动画队列保护。
- `localStorage` 读取失败时使用空进度，不阻断游戏。
- 诗卡配置缺失时结算仍可展示分数和星级，不应崩溃。
- 图片加载失败时使用角色颜色方块兜底。

## 测试策略

### 单元测试

- 棋盘匹配检测：横向、纵向、四连、五连、无匹配。
- 交换规则：相邻交换、非相邻不可交换、无匹配回弹。
- 消除和下落：空位补齐、行列坐标正确。
- 计分和星级：边界分数正确。
- 诗卡解锁：按主题、星级和已有进度正确合并。
- 收藏逻辑：收藏、取消收藏、重复收藏幂等。

### 手动验证

- 首页能进入酷跑、连连看、诗集。
- 四个主题均可进入连连看。
- 五个角色都能在三连时触发对应动画。
- 结算能展示并收藏诗卡。
- 刷新页面后收藏和最好成绩仍保留。
- 移动端宽度下按钮、棋盘、诗卡文字不重叠。

## 实施顺序建议

1. 抽出共享主题配置，保证酷跑与连连看可复用主题信息。
2. 增加乐园首页和入口流转。
3. 实现连连看纯逻辑模块和测试。
4. 实现 `Match3Scene` 棋盘渲染、交换、消除、补齐。
5. 接入五个角色方块和三连动画。
6. 接入四个主题关卡背景。
7. 实现分数、星级、主题诗库解锁和诗卡结算。
8. 实现诗集收藏与本地进度。
9. 进行移动端视觉和交互验证。

## 验收标准

- 用户打开应用先看到萌娃乐园首页，而不是直接进入酷跑。
- 用户可以从首页进入酷跑，并保持现有酷跑能力可用。
- 用户可以从首页进入萌娃连连看，选择四个主题之一开始游戏。
- 连连看支持相邻交换、横/竖三连、消除、下落、补齐和计分。
- 五个角色方块均存在，三连时触发对应角色动画。
- 一局结束后根据得分/星级解锁对应主题诗卡。
- 用户可以收藏已解锁诗卡，并在诗集中看到收藏状态。
- 朗读/跟读入口存在但不会承诺已实现能力。
- `npm.cmd run test` 和 `npm.cmd run build` 通过。
