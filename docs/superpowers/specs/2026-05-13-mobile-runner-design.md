# 移动端跑酷 H5 设计文档

## 项目目标

为小学生设计一款简单、好上手的移动端跑酷游戏。第一版优先验证核心跑酷手感和基础角色配置能力，先以手机浏览器可玩的 H5 形态交付，后续再根据效果迁移到微信小程序或原生应用。

## 平台形态

第一版采用移动端 H5。游戏可以在现代手机浏览器中运行，也可以在本地开发环境中预览和调试。实现时需要适当拆分游戏逻辑、配置数据和渲染表现，方便后续迁移到微信小程序。

## 核心玩法

游戏采用类似《地铁跑酷》《汤姆猫跑酷》的三跑道正向视角。角色会自动向前奔跑，玩家通过简单手势控制角色躲避障碍和收集奖励。

- 左右滑动：切换跑道。
- 向上滑动：跳跃，越过低矮障碍。
- 向下滑动：滑铲，通过高处障碍。
- 键盘方向键：用于桌面端调试和演示。

当角色撞到未正确躲避的障碍时，本局结束。结果界面展示得分，并提供一键重新开始。

## 第一版内容范围

第一版包含以下内容：

- 一个可玩的三跑道跑酷场景。
- 路障、高处横杆等基础障碍。
- 金币或星星类收集物。
- 距离和收集物共同构成的得分。
- 随时间逐步提升的游戏速度。
- 开始、角色配置、游戏中、暂停或结算等基础状态。
- 适合儿童的鼓励性结算文案。

第一版不包含登录、在线排行榜、付费道具、多人玩法、后端存储和微信专属 API。

## 角色配置

角色配置第一版保持轻量，重点是让孩子能快速看懂并产生参与感。玩家可以在开始前选择基础角色，并配置少量明显可见的外观部件。

- 基础角色预设。
- 上衣颜色。
- 帽子或背包配件。

角色配置采用数据驱动方式，后续可以继续增加角色、颜色和配件，不需要重写核心游戏循环。

## 视觉方向

整体视觉风格应明亮、圆润、友好，降低挫败感。界面文案避免使用强烈失败表达，重开流程要短。所有按钮和触控区域需要适配竖屏手机，保证小学生也能轻松点击和理解。

## 技术方案

第一版推荐使用 Phaser 3 实现 H5 原型。Phaser 3 适合 2D 游戏开发，能较好支持场景管理、动画节奏、输入处理、碰撞检测和资源管理。项目应采用轻量 Web 构建方式，支持本地开发和静态部署。

建议模块边界如下：

- 游戏启动和 Phaser 配置。
- 跑酷场景和游戏状态切换。
- 触控手势和键盘调试输入。
- 障碍物和收集物生成。
- 角色配置数据和角色渲染。
- 开始、配置、暂停、结算和重试界面。

## 测试与验收

验证内容包括：

- 本地构建成功。
- 本地开发服务器可以打开游戏。
- 移动端视口下界面不溢出、不重叠。
- 触控手势和键盘方向键可用。
- 碰撞、计分、收集和重开流程正常。
- 角色配置能明显影响游戏中的角色外观。

完成前需要通过浏览器移动端视口检查，必要时截屏确认画面比例、文字和按钮布局。

## 验收标准

- 用户可以在移动端尺寸下打开 H5 游戏，并通过一次点击开始游玩。
- 角色可以在三条跑道之间移动，并支持左右、跳跃、滑铲动作。
- 游戏过程中会出现障碍物和收集物。
- 撞到障碍后游戏结束，并展示得分和重新开始入口。
- 开始前可以进行基础角色配置。
- 项目提供本地运行说明，方便继续开发和演示。
