# 主题关卡与生命值设计

## 目标

为当前移动端跑酷 H5 增加下一版可试玩 MVP：让孩子能在一次跑酷中体验校园、商场、动物园、游乐园四个有辨识度的场景，同时把“碰到障碍物立即结束”改成更友好的生命值机制。

这一版重点是先做出能在手机尺寸上试玩、能给孩子验证手感的版本。代码结构需要为后续迁移到微信小程序或小游戏保留空间。

## 范围

本次迭代包含：

- 四个主题关卡环境：校园、商场、动物园、游乐园。
- 每个主题有独立的视觉地标和障碍物风格。
- 每个主题有独立背景音乐入口，并支持跑酷中切换音乐。
- 新增生命值系统，障碍物先扣血，不再立即结束游戏。
- 新增受击反馈：音效、角色震动、闪烁、短暂无敌。
- 为主题切换和生命值逻辑补充纯函数测试。

本次迭代不要求最终商业音频素材、完整精灵图资源、关卡选择页、账号或成长系统。这些等孩子试玩确认核心循环后再扩展。

## 推荐方案

采用“配置驱动的主题关卡”方案，而不是为每个地点创建独立 Phaser Scene。

`RunnerScene` 继续负责通用跑酷循环，新增配置模块描述每个主题的颜色、地标、障碍物样式和音乐 key。这样做的好处是：

- 当前原型迭代更快。
- 增加新场景时主要改配置。
- 后续迁移小程序时，关卡数据和游戏逻辑更容易复用。

## 关卡主题

游戏在一次跑酷中按距离自动切换四个主题，让孩子不用进入复杂菜单也能看到场景变化。

- 校园：教学楼、操场跑道、图书馆标识；障碍物可用书包、路障、栏杆。
- 商场：游戏厅灯牌、餐厅门店、电影院招牌；障碍物可用购物袋、餐桌、爆米花桶。
- 动物园：动物区域指示牌、围栏、树木、动物剪影；障碍物可用木栅栏、草丛、饲料桶。
- 游乐园：摩天轮、过山车轨道、旋转木马元素；障碍物可用排队护栏、气球摊、小推车。整体氛围参考长隆游乐园，偏明亮、热闹、亲子友好。

每个主题必须在首屏就能被识别出来，不能只靠角落里的小文字说明。

## 音频设计

每个主题都有独立的 `musicKey`。初版可以先使用代码生成的轻量音色或临时音频，只要架构上把它当成可替换资源处理。

音频管理规则：

- 跑酷开始时播放当前主题背景音乐。
- 场景主题切换时切换背景音乐，可先用停止旧音乐、播放新音乐的方式实现。
- 玩家受击时播放短促受击音效。
- 游戏结束或回到选择界面时停止背景音乐。

后续替换真实 `mp3` 或 `ogg` 素材时，只应修改资源路径和主题配置，不应改碰撞逻辑或关卡推进逻辑。

## 生命值与受击反馈

每局开始时玩家有 3 点生命值。

障碍物碰撞规则：

- 如果障碍物被正确躲避，例如跳过低障碍或下滑躲过高障碍，不扣血。
- 如果玩家当前可受击，生命值减 1。
- 被撞到的障碍物需要销毁或标记为已处理，避免同一个障碍连续扣血。
- 玩家进入约 1000 毫秒短暂无敌期。
- 角色播放闪烁、左右震动、短暂后退等反馈。
- 播放受击音效。
- 生命值降到 0 时才结束游戏并进入结算界面。

HUD 在跑酷过程中始终显示生命值。初版可以用心形或分段血条。后续可以继续扩展低生命状态下的角色表情、动作变化、喘气音效或警示音。

## 代码结构

新增或修改以下模块：

- `src/game/levels.ts`：定义主题 id、显示名、距离阈值、色彩、地标描述、障碍物描述和音乐 key。
- `src/game/health.ts`：提供生命值初始化、扣血、死亡判断、无敌期判断等纯函数。
- `src/game/levels.test.ts`：测试距离到主题的映射，以及超过最后主题后的兜底逻辑。
- `src/game/health.test.ts`：测试扣血、生命值下限、无敌期和死亡状态。
- `src/game/RunnerScene.ts`：读取主题和生命值配置，绘制主题背景，切换音乐，更新生命值 HUD，把障碍碰撞从立即结束改为扣血。

初版主题绘制可以先留在 `RunnerScene` 中。如果文件变得难读，再拆出 `src/game/themeDrawing.ts`。

## 用户体验

这版体验目标是“犯错有反馈，但不马上惩罚”。

- 单次失误只扣血，不直接结束。
- 玩家能通过音效和动画明确知道自己被撞到了。
- 生命值变化要清楚，但不能遮挡操作。
- 场景切换要像进入了一个新地点，有明显地标和音乐变化。
- 只有生命值归零时才显示结算界面。

## 测试与验收

自动化测试覆盖纯逻辑：

- 不同距离能映射到正确主题。
- 超过最后主题距离后仍停留在最后主题。
- 生命值初始为 3。
- 每次有效受击生命值减 1。
- 生命值不会低于 0。
- 无敌期内再次受击不扣血。
- 生命值为 0 时判定失败。

手动验收覆盖：

- 390 x 844 左右的手机视口下界面正常。
- 角色选择界面仍然完整可点。
- 跑酷过程中能看到四个主题变化。
- 每个主题的视觉地标清晰。
- 每个主题都有背景音乐切换效果。
- 撞到障碍物先扣血，不立即结束。
- 受击反馈可见、可听。
- 生命值为 0 后进入结算界面。

## 小程序迁移注意事项

为了后续更容易迁移到微信小程序或小游戏：

- 主题数据保持可序列化，避免混入 Phaser 对象。
- 生命值和关卡推进规则保持纯函数，并配套测试。
- 音频 key 不要直接写死在碰撞逻辑里。
- 浏览器特有 API 尽量只放在 Phaser 启动层或音频适配层。

这样当前 H5 原型可以快速给孩子试玩，后续迁移时也能复用核心规则和大部分配置。
