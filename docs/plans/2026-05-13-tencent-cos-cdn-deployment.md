# 腾讯云 COS + CDN 部署方案

## 目标

把《萌娃酷跑》部署成一个国内访问更稳定的 H5 试玩链接，主要面向微信内置浏览器、手机浏览器和朋友转发场景。

GitHub Pages 保留为开发预览和备份链接。腾讯云 COS + CDN 作为国内朋友试玩的主入口。

## 费用说明

腾讯云 COS + CDN 不是完全免费方案，通常按量计费。

主要费用包括：

- COS 存储费用：`dist` 静态文件占用的存储空间。
- COS 请求费用：用户访问文件产生的请求次数。
- CDN 下行流量费用：用户从 CDN 节点下载 HTML、JS、CSS、图片等资源。
- CDN 回源到 COS 的相关费用：CDN 缓存未命中时回源读取 COS。

当前游戏体积较小，试玩阶段流量通常很低。建议仍然设置费用预算和告警，避免链接传播超出预期后产生额外费用。

## 推荐架构

```text
用户微信/浏览器
  -> 自定义域名或 CDN 域名
  -> 腾讯云 CDN
  -> 腾讯云 COS 静态网站源站
  -> dist 静态文件
```

## 前置准备

- 腾讯云账号。
- 已实名。
- 一个 COS Bucket。
- 一个可选域名。

说明：

- 没有域名也能先用 COS 或 CDN 默认域名测试。
- 如果使用自己的国内域名并接入国内 CDN，通常需要完成 ICP 备案。
- 长期给朋友分享，建议使用自己的域名和 HTTPS。

## 第一步：本地构建

在项目根目录执行：

```bash
npm install
npm.cmd run test
npm.cmd run build
```

构建产物在：

```text
dist/
```

上传到 COS 的是 `dist` 目录里的内容，不是上传整个项目源码。

## 第二步：创建 COS Bucket

在腾讯云控制台：

1. 进入 **对象存储 COS**。
2. 创建 Bucket。
3. 地域建议选择离主要用户近的地域，例如：
   - 华南用户多：广州
   - 华东用户多：上海
   - 华北用户多：北京
4. 访问权限建议：
   - 测试阶段可选择公有读私有写。
   - 如果后续使用 CDN，也可以按腾讯云推荐配置源站访问权限。

Bucket 命名示例：

```text
mengwa-runner-playtest
```

## 第三步：上传 dist 文件

进入 Bucket 文件管理：

1. 打开 `dist` 目录。
2. 上传 `dist` 目录中的所有文件和子目录。
3. 确保 `index.html` 位于 Bucket 根目录。

正确结构示例：

```text
/
  index.html
  assets/
    index-xxxx.css
    index-xxxx.js
```

错误结构示例：

```text
/
  dist/
    index.html
```

如果出现这种结构，访问时会找不到首页。

## 第四步：开启静态网站

在 COS Bucket 设置中：

1. 找到 **基础配置** 或 **静态网站**。
2. 开启静态网站功能。
3. 首页文档设置为：

```text
index.html
```

4. 错误文档可先设置为：

```text
index.html
```

当前游戏是单页应用，错误文档设为 `index.html` 更稳。

开启后，先用 COS 提供的静态网站访问地址测试。

## 第五步：接入 CDN

在腾讯云 CDN 控制台：

1. 添加域名。
2. 加速类型选择：

```text
网页小文件
```

3. 源站类型选择：

```text
COS 源
```

4. 源站选择刚才创建的 Bucket。
5. 加速区域：
   - 国内朋友试玩：选择中国境内。
   - 如果没有备案域名，可能无法使用中国境内 CDN，需要先备案或使用可用的测试域名。

## 第六步：缓存策略

建议缓存策略：

| 文件 | 缓存时间 | 原因 |
| --- | --- | --- |
| `index.html` | 1 分钟到 5 分钟 | 方便快速发布新版本 |
| `assets/*.js` | 30 天 | Vite 文件名带 hash，可长缓存 |
| `assets/*.css` | 30 天 | Vite 文件名带 hash，可长缓存 |
| 图片/音频 | 30 天 | 素材文件更新频率低 |

如果腾讯云控制台支持按路径配置，可参考：

```text
/index.html        300 秒
/assets/*          2592000 秒
```

每次发布新版本后，建议刷新：

```text
/index.html
```

必要时再刷新：

```text
/assets/*
```

## 第七步：HTTPS

建议启用 HTTPS。

如果使用自定义域名：

1. 在腾讯云申请免费 SSL 证书，或上传已有证书。
2. 在 CDN 域名配置中绑定证书。
3. 开启 HTTP 自动跳转 HTTPS。

微信内置浏览器里分享游戏，HTTPS 是基本要求。

## 第八步：访问测试

至少测试这些场景：

- 微信内置浏览器打开。
- iPhone Safari 打开。
- Android 手机浏览器打开。
- 首屏是否能快速看到“萌娃酷跑 正在加载游戏”。
- 是否能进入角色/关卡选择。
- 是否能开始一局、暂停、继续、结算。
- 音效是否需要用户点击后才播放。

## 第九步：更新发布流程

每次发新版本：

1. 本地验证：

```bash
npm.cmd run test
npm.cmd run build
```

2. 上传新的 `dist` 内容到 COS。
3. 覆盖旧文件。
4. 刷新 CDN 缓存：

```text
/index.html
```

5. 手机微信打开链接，确认版本更新。

## 第十步：费用与安全建议

建议开启：

- 费用预算提醒。
- CDN 流量告警。
- COS 请求量告警。
- 防盗链，后续有正式域名后再配置。

试玩阶段建议先小范围分享，不要直接公开大范围传播。

## 回滚方案

推荐每次发布前备份上一版 `dist`：

```text
release/
  v0.2.0/
  v0.2.1/
```

如果新版本有问题：

1. 将上一版 `dist` 文件重新上传到 Bucket 根目录。
2. 刷新 CDN：

```text
/index.html
/assets/*
```

3. 手机端重新打开确认。

## 腾讯云方案适合当前项目的原因

- 当前项目是纯静态 H5，适合 COS 托管。
- 朋友主要在国内，CDN 访问稳定性通常比 GitHub Pages 更好。
- 后续如果接微信生态，腾讯云链路更自然。
- 成本随访问量增长，试玩阶段可控。

## 下一步

建议先完成：

1. 创建 COS Bucket。
2. 手动上传一次 `dist`。
3. 用 COS 静态网站域名测试。
4. 如果速度明显改善，再接 CDN 和自定义域名。

等手动流程跑通后，再考虑做自动上传脚本或 GitHub Actions 自动发布到 COS。
